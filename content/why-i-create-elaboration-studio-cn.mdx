## 🙋🏻‍♂️ 问题的定义

Elaboration Studio 致力于通过人工智能技术帮助个人更好地「演绎观点」和「解决问题」。因此它主要应对以下几个挑战：

- 如何有效地整理与当前问题相关的复杂信息？
- 如何利用现有的先进技术工具和理论来最大化发挥人工智能的效能？
- 如何构建并不断优化一个思维系统，使其能够适应新信息和新挑战，确保在解决问题方法上的持续改进和创新？

为了解决上述问题，我计划在一个统一的平台上无缝地整合“工作空间”、“文档管理”、“AI助手”和“聊天”功能。接下来，我会逐一介绍这些功能及其设计理念。

## 🧠 在 Workspace 中汇聚上下文

正确地准备上下文信息，特别是与用户相关的信息，对于改善整体用户体验非常重要。组织这些信息可以让人工智能更好地理解和响应用户的查询，同时保持内容的相关性和准确性。这确保了人工智能提供的答案能够满足个人的需求。

在 Elaboration Studio 中，我们使用“工作空间”来整理和管理所有相关信息，让用户能够在单一且流畅的环境中高效地访问和使用这些信息。

工作空间集成了多种工具和功能，增强了用户与人工智能的交互体验，提供了对文档、文件、助手和聊天功能的一站式访问。这种整合创造了一个连贯的工作环境，在这里用户可以高效地管理资源，无缝协作，并利用人工智能的能力来解决他们特有的需求和问题。

“文档 + 助手 + 聊天 + 资源”都在一个工作空间里。

在我们的工作空间中，我们力求整合几个核心元素：

- **文档管理**：这些文档结构类似 Notion 块，采用模块化组件，方便灵活组织和整合各类内容。
- **智能助手**：这些专门的人工智能助手根据工作空间内的上下文和信息为用户提供定制的答案和见解。
- **历史对话**：此功能让用户可以与人工智能讨论历史事件，提供用户选定的上下文和见解，并储存记忆。
- **资源管理**：通过将相关文件如 PDF 和多媒体文件作为资源来组织，确保了它们易于访问和检索。

为了在工作空间内有效组织这些元素，并便于用户导航和获取相关信息，我们必须确保这一过程足够顺畅。这样可以促进平台上的高效协作。在 Elaboration Studio 中，我们计划使用图结构或其它非线性的结构来展示这些不同类型的资源。

接下来，我们将讨论信息组织的核心部分：**文档管理**。它们作为工作空间的基础构建模块，让用户可以创建、编辑和合作处理结构化的文档，这些文档能够与人工智能驱动的洞察和上下文信息无缝结合。

## 💠 Elaboration Docs

> 我们不是笔记应用。我们从 Notion 学习，但是以一种更加简单优雅的方式。专注于写作的简便性。专注于高度定制的人工智能集成。

我们不仅仅是一个笔记应用。虽然我们借鉴了 Notion 的一些理念，但我们提供的是更为简单和优雅的体验。我们注重写作的简便性，并致力于高度定制化的人工智能集成。Elaboration 文档提供了一个干净、简单且优雅的编辑界面，使得与其他应用程序（如 Arc 浏览器）的合作变得轻松自如。

在 Elaboration 文档中，我们借鉴了 Notion 的模块化架构和内容写作体验，使用户能够以灵活且直观的方式创建和组织内容，并实现了人工智能驱动功能的无缝融合，以提升写作和协作的效果。

- **人工智能辅助的文章写作**：人工智能即时补全技术已经彻底改变了我们的写作方式。通过实时建议（基于文章和上下文），大大提高了写作效率。或者，您也可以通过快捷键手动触发人工智能补全，提供与人工智能协作的良好写作体验。
- **与人工智能对话的文档**：用户可以轻松地通过文档中的提示或直接聊天与人工智能互动。
- **基于丰富上下文的问答**：利用多种来源如文档、工具和用户互动，提供全面且准确的答案。
- **丰富的模块类型**：我们预设了一些人工智能驱动的模块来协助您的内容创作。
  - **金融查询**：您可以提问以获取金融领域相关的数据，并让人工智能帮助您进行分析。
  - **提示器**：您可以在文档中插入提示模块，并直接在文档中提出提示，将本文档中的内容或引用的相关内容作为上下文提供给人工智能。
  - **阐述者**：提供一种方式让您直接在文档中与预建的人工智能代理互动。
  - **规划者**：您可以插入这个模块作为一个中心规划者，接受当前文档作为上下文，并根据您的问题或目标为您挑选阐述者并生成阐述者模块。将这些模块插入文档中，您可以使用上述代理来打造您的成果。
  - **思考助手** → 在模块中进行复杂问题求解的长时间轮询会话，以协作的方式解决复杂挑战，利用人工智能的能力生成见解，促进讨论，并实时改进解决方案。

凭借这些特性，Elaboration 文档为用户提供了直观高效的写作体验，允许无缝集成人工智能驱动的建议和基于上下文的问答，从而提高文档的整体质量和清晰度。这正是我们用人工智能的方式来写作文档并存储重要信息的方法。除了文档之外，我们还可以使用多种其他资源来增强我们的创作流程，比如多媒体文件、模板和策划的内容库，以支持多样化的项目需求并促进协作创新。

## 🗃️ Elaboration Resources

> 所有的工作都围绕 RAG 展开！

在精心编写文档之后，将现有内容整合进 Elaboration 资源中，便于访问和参考非常重要。这样做可以确保所有相关信息都能用来增强详细说明的过程。在 Elaboration Studio 中，我们将支持以下文件类型：

- 文档：Markdown 文本、PDFs 和 Microsoft Office 文档是最常用的创建和分享结构化信息的格式。这些格式使得用户可以轻松协作，并在 Elaboration Studio 中访问各种类型的内容。
- 媒体：图像、音频和视频格式有助于创建多模态的内容，通过多种方式与信息和想法互动，提升了整体用户体验。
- 数据库连接器：我们将支持连接到远程数据库，并与 Notion 和 Airtable 平台的集成。利用类似于 NL2SQL 的技术，用户可以使用自然语言处理无缝查询和操作来自不同来源的数据。这个功能将带来更加直观高效的数据管理体验，让用户可以从连接的数据库中获取相关的洞察和分析，从而基于实时数据分析做出明智的决策。
通过 Elaboration 资源，用户可以轻松访问各种类型的文档和媒体，确保所有必要的信息都触手可及，便于有效协作和发展创意。

接下来，我们将讨论作为代理的 Elaborators，用户可以创建定制化的人工智能助手，以帮助他们解决问题和提供见解。这些 Elaborators 将在工作空间中提供定制化的服务，以满足用户的特定需求和目标。

## 🪄 Elaborators

> 代理 = 提示工程 + ReAct + CoT + Mesh + …

用户可以轻松创建符合自己特定需求的代理（Elaborator），从而在各类任务中获得个性化的协助，提升整体工作效率。

eStudio 计划以以下方式设计和提供代理创建工具：

- **工作区元素作为上下文**：如前所述，工作区作为基本要素，提供了用户与 AI Elaborators 有效交互所需的上下文，确保信息有序且易于访问，从而促进合作和解决问题。
  - Elaboration 文档：用户精心编制的信息。
  - 资源：文件、数据等。
  - …
- **代理作为工具**：通过利用 LLM 的 ReAct 模式，用户可以将 Elaborators 作为强大的工具来增强解决问题的能力，提供针对特定需求的实时见解和建议。
- **代理选择器和 Elaborators 选择器**（问题 → 选定代理）：允许用户轻松选择最适合其特定任务的人工智能代理，从而提高所提供的帮助的效率和相关性。
- **代理记忆**：我们可以根据用户的互动和特定项目的需要动态调整每个 Elaborator 的能力和知识，无论是私人还是公共场景。

最后，对于如何与这些精心制作的 Elaborators 互动，我们可以采用：

- 在聊天窗口中通过对话（LUI）驱动
  - **直接聊天**通过 `/chat` API 开始与 Elaborators 的对话，使用户能够实时接收定制的帮助和见解。
  - **切换 Elaborator** 聊天，在保持相同上下文/消息列表的情况下，切换 Elaborator 聊天可以让用户保持对话和想法的一致性，确保在不同 AI 代理间转换时体验顺畅。
- 在侧边文档中聊天，自然地与 Elaboration 文档交互。
- 带有 AI 提示的内联块。

未来，随着 AI 大型语言模型成本下降且性能更强，我们可以借鉴 AutoGPT 的概念作为“代理网格”/“代理群集”（由 OpenAI 提出），创建一个专门的 Elaborators 网络，协同解决复杂任务，利用它们独特的优势和见解来增强 Elaboration Studio 中的问题解决和创新能力。

## ✨ 结语
